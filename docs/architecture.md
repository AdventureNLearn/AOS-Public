# AOS-Public Architecture Overview

## High-Level Design

The Adventure OS (AOS) system is built around a small set of powerful, composable layers:

### 1. Core Orchestration Layer
- Single source of truth for complex work
- Program 0 Compression + 4-Agent model

### 2. Foundational Verification Layer
- Evidence vs Inference discipline
- Mission Spine enforcement
- Anti-pattern detection

### 3. Reasoning & Analysis Layer
- Deep cross-domain reasoning
- Sovereign Lens (structured multi-module overlay)

### 4. Protocol Layer
- Shatter Protocol (prompt illusion breaking)
- Ops Hardening Framework
- Influence & Organizational Mapping Framework

## Design Principles

- **Modular**: Components can be used independently or composed.
- **Observable**: All routing and decisions should be explicit.
- **Gate-First**: Critical verification happens before progression.
