# Worked Example: Shatter + Ops Hardening Framework

This example shows how to use the Shatter Protocol together with the Ops Hardening Framework on an institutional process.

## Query
"Analyze the public comment and review process for a major federal agency rulemaking. Identify potential points of failure, narrative control, or process hardening opportunities."

## Step 1: Apply Shatter Protocol
Shatter. Shatter.

Key Revelations:
- The dominant narrative presents the public comment process as open, transparent, and responsive.
- There are observable patterns of comment volume management, selective response, and framing that can limit meaningful public input.
- Process design (comment periods, docket systems, response formatting) can function as soft control mechanisms.

## Step 2: Apply Ops Hardening Framework
Using the gate-first and process resilience lens:

**Key Process Weak Points Identified:**
- Short or poorly timed comment periods
- Lack of structured response to substantive comments
- Opaque internal review processes for which comments receive attention
- Limited feedback loops back to the public

**Hardening Recommendations:**
- Implement clear gates for comment categorization and response
- Create auditable trails for how comments influence final rules
- Add early detection mechanisms for coordinated or low-quality comment campaigns
- Build continuous improvement loops based on public feedback quality

## Final Synthesized Output
While the public comment process appears open, structural and procedural weaknesses can reduce its effectiveness as a genuine accountability mechanism. Applying hardening principles (early gates, auditability, and feedback loops) could significantly improve both resilience and public trust.

Key Takeaway:
Shatter revealed potential control layers in the process narrative. The Ops Hardening Framework then provided concrete structural improvements to make the process more robust and accountable.
