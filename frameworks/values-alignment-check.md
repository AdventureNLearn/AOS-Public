# Values Alignment Check (Optional)

A lightweight, optional pattern for reviewing outputs against principles the user defines.

## Purpose

When doing high-stakes, public-facing, or value-sensitive work, it can be useful to explicitly check whether the output aligns with chosen principles before final delivery.

## How to Use

1. Define the values or principles that matter for the current task.
2. Review the proposed output against those principles.
3. Revise or flag any parts that appear to drift from the defined principles.
4. Proceed only when alignment is acceptable (or when you consciously choose to override with justification).

## Example Principles

Users should define their own. Common examples include:
- Evidence is clearly labeled and traceable
- Claims are proportionate to available support
- Language remains direct and avoids unnecessary hedging
- Output maintains independence from external narrative pressure
- Work stays consistent with previously stated goals or scope

## When to Use

This pattern is most useful:
- Before publishing public content or frameworks
- Before final delivery on sensitive or high-stakes topics
- As a lightweight final review step in any workflow

## Integration

Can be used:
- As a standalone final check
- Combined with evidence labeling
- As an optional addition to any existing coordination pattern

This is an **optional** pattern. The user defines the values. There is no pre-set or enforced list.
