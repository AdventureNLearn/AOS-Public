# Core Orchestration Pattern

A reusable workflow for coordinating complex, multi-step reasoning and build tasks.

## Overview

This pattern combines four complementary techniques:

- **Program 0** — Compress the task before starting
- **4-Agent Coordination** — Clear roles with structured handoffs
- **Didymus-Style Verification** — Review for consistency and evidence quality
- **Phoenix-Style Feedback** — Capture lessons for future improvement

## When to Use

Use this pattern when working on tasks that involve multiple steps, require careful reasoning, or benefit from verification before final delivery.

## Core Components

### 1. Task Compression (Program 0)
Reduce the task to its essential objectives and smallest starting point before delegating or expanding work.

### 2. 4-Agent Coordination
Divide work across four roles:
- Coordinator (overall flow and synthesis)
- Reasoning & Evidence
- Verification
- Execution & Delivery

### 3. Verification Gate
Before finalizing output, check that claims are properly evidenced and logically consistent.

### 4. Recursive Feedback
After completing work, record what worked well and what could be improved next time.

## Observable Handoffs

When moving work between roles or skills, use a short structured note so the flow remains clear and traceable.

## Benefits

This pattern helps reduce errors, improve traceability, and create compounding improvement across tasks and sessions.
